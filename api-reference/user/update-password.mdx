---
title: 'Update Password'
api: 'POST /api/v1/user/update_password'
description: 'Set new password after reset'
---

## Endpoint

```
POST /api/v1/user/update_password
```



## Headers

<ParamField header="X-API-Token" type="string" required>
  Your API access token. You can also set this globally in the API playground.
</ParamField>

<ParamField header="platform" type="string" required>
  The platform making the request: `WEB`, `IOS`, or `ANDROID`.
</ParamField>

<ParamField header="Authorization" type="string" required placeholder="Bearer {token}">
  User authentication token (Bearer token received after signin).
</ParamField>
## Request Body

<ParamField body="token" type="string" required>
  Password reset token from email
</ParamField>

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="password" type="string" required>
  New password (minimum 8 characters)
</ParamField>

<ParamField body="password_confirmation" type="string" required>
  Password confirmation (must match password)
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

## Password Requirements

<AccordionGroup>
  <Accordion title="Minimum Length">
    Password must be at least 8 characters long
  </Accordion>
  <Accordion title="Confirmation">
    Password and password_confirmation must match exactly
  </Accordion>
  <Accordion title="Recommendations">
    - Use a mix of uppercase and lowercase letters
    - Include numbers and special characters
    - Avoid common words or patterns
    - Don't reuse old passwords
  </Accordion>
</AccordionGroup>

## Notes

- Token is consumed after successful password update
- All existing sessions are invalidated after password change
- User must sign in again with the new password
- Token expires after 60 minutes
- After successful reset, redirect user to sign in page
