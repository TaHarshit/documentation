## Endpoint

```
POST /api/v1/user/qrcode_signin
```



## Headers

<ParamField header="X-API-Token" type="string" required>
  Your API access token. You can also set this globally in the API playground.
</ParamField>

<ParamField header="platform" type="string" required>
  The platform making the request: `WEB`, `IOS`, or `ANDROID`.
</ParamField>
## Request Body

<ParamField body="qr_code" type="string" required>
  QR code data/token
</ParamField>

<ParamField body="device_token" type="string">
  Device token for push notifications
</ParamField>

<ParamField body="platform" type="string" required>
  Platform identifier: IOS, ANDROID, or WEB
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="Authentication Data">
    <ResponseField name="api_token" type="string">
      Bearer token for authentication
    </ResponseField>
    <ResponseField name="user" type="object">
      Complete user profile information
    </ResponseField>
  </Expandable>
</ResponseField>

## QR Code Authentication Flow

<Steps>
  <Step title="Generate QR Code">
    User generates a QR code from their authenticated session
  </Step>
  <Step title="Scan QR Code">
    Another device scans the QR code to extract the authentication token
  </Step>
  <Step title="Authenticate">
    Device sends the QR code data to this endpoint
  </Step>
  <Step title="Receive Token">
    Device receives API token and can access the account
  </Step>
</Steps>

## Notes

- QR codes have a limited validity period
- QR code can only be used once for security
- Useful for quick authentication on new devices
- Device token is optional but recommended for push notifications
- Store the returned API token securely
