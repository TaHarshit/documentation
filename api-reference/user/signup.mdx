## Endpoint

```
POST /api/v1/user/signup
```



## Headers

<ParamField header="X-API-Token" type="string" required>
  Your API access token. You can also set this globally in the API playground.
</ParamField>

<ParamField header="platform" type="string" required>
  The platform making the request: `WEB`, `IOS`, or `ANDROID`.
</ParamField>
## Request Body

<ParamField body="name" type="string" required>
  User's full name
</ParamField>

<ParamField body="email" type="string" required>
  Must be a unique and valid email address.
</ParamField>

<ParamField body="phone_no" type="string" required>
  User's phone number
</ParamField>

<ParamField body="password" type="string" required>
  User's password (minimum 8 characters)
</ParamField>

<ParamField body="country_code" type="string" required>
  Country code for phone number (e.g., "91" for India, "1" for US)
</ParamField>

<ParamField body="profile_image" type="file">
  User's profile image (optional)
</ParamField>

<ParamField body="device_token" type="string">
  Device token for push notifications
</ParamField>

<ParamField body="platform" type="string" required>
  Platform identifier: IOS, ANDROID, or WEB
</ParamField>

<ParamField body="company_name" type="string">
  Required if reg_as_company is 0.
</ParamField>

<ParamField body="reg_as_company" type="integer" required>
  0 for Company User, 1 for Independent User.
</ParamField>

<ParamField body="own_company_name" type="string">
  New company name (required if reg_as_company = 1)
</ParamField>

## Notes

- Email must be unique across all users
- Password must be at least 8 characters
- Profile image is optional but recommended
- Device token is used for push notifications
- Company registration:
  - Set `reg_as_company=1` to create your own company
  - Set `reg_as_company=0` to join an existing company (requires `company_name`)
- After successful signup, use the returned `api_token` for authenticated requests
